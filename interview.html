Linux & DevOps Interview Questions (L2 & L3)

1. What will you do if a server is slow?
Check CPU, memory, disk, network.
top free -m iostat vmstat
2. Difference between soft link and hard link?
Soft link points to file path, hard link points to inode. Hard link points to the same inode, so even if the original file is deleted, data remains. Soft link is a pointer to the file path, so it breaks if the original file is removed
3. How to check which process is using a port?
ss -tulnp lsof -i :80
4. What happens when disk is full?
System may crash, logs stop, services fail.
5. How to find large files?
find / -type f -size +1G
6. What is load average?
Load average shows the number of processes waiting for CPU or in runnable state. load average: 0.50, 1.20, 2.00
7. Service not starting?
Check status, logs, config.
systemctl status journalctl -xe
8. What is Docker?
Docker is a containerization platform used to build, package, and run applications in containers. # Pull image docker pull nginx # Run container docker run -d -p 80:80 nginx
9. Pod CrashLoopBackOff?
kubectl logs kubectl describe pod 🔹 Common Causes ❌ Application crash (bug) ❌ Wrong command in container ❌ Missing environment variables ❌ ConfigMap/Secret issue ❌ Port already in use ❌ Dependency (DB/API) not reachable
10. Website is down?
Check server → service → port → logs → network
11. Service not starting — how do you troubleshoot?
Check service status and logs:
systemctl status nginx journalctl -xe
Example: Port already in use → change port or stop conflicting service.
12. Disk full — what will you do?
Check disk usage:
df -h du -sh *
Example: Logs consuming space → delete or rotate logs.
13. High CPU usage — how to find the cause?
Identify process:
top ps aux --sort=-%cpu
Example: Java process high CPU → restart or optimize application.
14. SSH not working — what will you check?
Check service and port:
systemctl status sshd ss -tulnp | grep 22
Example: Firewall blocking → allow port 22.
15. Website not accessible — troubleshooting steps?
Check service and connectivity:
systemctl status nginx curl localhost ss -tulnp
Example: Service down → start service.
16. File deleted accidentally — how to recover?
Check backups or restore snapshot.
Example: Restore from backup or LVM snapshot.
17. User cannot login — what will you check?
Check user status:
id username passwd -S username cat /etc/passwd
Example: Account locked → unlock user.
18. Memory high usage — how to troubleshoot?
Check memory usage:
free -m top
Example: Process consuming memory → restart or optimize.
19. Cron job not running — debug steps?
Check cron service and logs:
crontab -l systemctl status crond tail -f /var/log/cron
Example: Wrong schedule → fix cron timing.
20. Permission denied error — how to fix?
Check permissions:
ls -l chmod 755 file chown user:user file
Example: Incorrect ownership → fix using chown.

Advanced L3 Scenario Questions (21–30)

21. Database suddenly slow in production — how do you troubleshoot?
Check DB and system bottlenecks:
top iostat -x 1 netstat -ant | grep 3306
Then analyze slow queries in DB logs.
Example: High disk latency → slow DB queries → move to SSD or optimize queries.
22. Kubernetes node NotReady — what will you check?
Check node and kubelet status:
kubectl describe node systemctl status kubelet journalctl -u kubelet
Example: Kubelet stopped → restart kubelet → node becomes Ready.
23. Application working locally but failing in server — why?
Check environment differences:
env printenv ldd binary
Example: Missing dependency or wrong ENV variable.
24. Too many open files error — how to fix?
Check limits:
ulimit -n lsof | wc -l
Increase limit in /etc/security/limits.conf.
Example: Increase from 1024 → 65535 for high traffic apps.
25. Kubernetes service not accessible externally — debug steps?
Check service type and endpoints:
kubectl get svc kubectl get endpoints kubectl describe svc
Example: Wrong selector → no endpoints → fix labels.
26. High swap usage — what does it indicate?
Check memory pressure:
free -m vmstat 1
Example: Low RAM → system using swap → add RAM or optimize memory usage.
27. SSL certificate expired — impact and fix?
Check certificate:
openssl x509 -in cert.pem -noout -dates
Renew and reload service.
Example: Expired cert → users see HTTPS error → renew via certbot.
28. Container exits immediately after start — why?
Check logs and entrypoint:
docker logs container_id kubectl logs pod
Example: Wrong CMD → container exits → fix Dockerfile.
29. Network connectivity issue between servers — debug?
Check connectivity path:
ping traceroute telnet ip port
Example: Firewall blocking port → open port → connectivity restored.
30. Logs consuming huge disk space — what will you do?
Identify large logs:
du -sh /var/log/*
Apply log rotation:
logrotate -f /etc/logrotate.conf
Example: Large logs → rotate and compress → free space.
31. Server suddenly rebooted – what will you check?
Check logs
last reboot journalctl -xb -1 dmesg | less
Root cause: OOM / kernel panic / hardware.
32. Application slow at specific time?
Check cron
crontab -l ls /etc/cron*
Likely scheduled job causing load.
33. CPU 100% but no obvious process?
top -o %CPU ps -eo pid,cmd,%cpu --sort=-%cpu | head
Check hidden/background processes.
34. Disk full but no large files?
lsof | grep deleted
Deleted files held by process → restart it.
35. App works locally but not externally?
ss -tulnp firewall-cmd --list-all
Check firewall & binding (0.0.0.0).
36. K8s pod running but app not accessible?
kubectl get svc kubectl get endpoints
Service/label mismatch.
37. One pod getting all traffic?
Check service/ingress config → load balancing or stickiness issue.
38. Jenkins build slow?
top df -h
Check CPU/disk/workspace cleanup.
39. Docker container exits immediately?
docker logs container_id
Wrong CMD or app crash.
40. High memory but no process?
free -m slabtop
Kernel/cache usage.
41. SSH login slow?
DNS delay → disable UseDNS
cat /etc/resolv.conf
42. LB giving 502 error?
tail -f /var/log/nginx/error.log
Backend down/misconfig.
43. Logs not generating?
df -h ls -l /var/log
Disk full/permission issue.
44. K8s node NotReady?
kubectl describe node systemctl status kubelet
Kubelet/resource issue.
45. Network latency issue?
ping traceroute mtr
Network congestion/path issue.
46. DB connection timeout?
telnet db_ip 3306 systemctl status mysql
DB down/firewall.
47. Swap usage high?
free -m vmstat 1
Low RAM/memory leak.
48. Logs growing fast?
du -sh /var/log/*
Use logrotate.
49. K8s deployment failed?
kubectl rollout status deployment kubectl describe pod
Image/config issue.
50. SSL not working?
openssl s_client -connect domain:443
Expired/misconfigured cert.

LVM Interview Questions & Scenarios

LVM Q1: What is LVM and why use it?
LVM allows flexible disk management, resizing, snapshots, and combining multiple disks.
LVM Q2: Main components of LVM?
PV → Physical disk
VG → Storage pool
LV → Logical partition
LVM Q3: Create and mount LVM?
pvcreate /dev/sdb vgcreate vg_data /dev/sdb lvcreate -L 10G -n lv_data vg_data mkfs.ext4 /dev/vg_data/lv_data mount /dev/vg_data/lv_data /mnt
LVM Q4: Extend logical volume?
lvextend -L +5G /dev/vg_data/lv_data resize2fs /dev/vg_data/lv_data
Use xfs_growfs for XFS.
LVM Q5: /home running out of space?
pvcreate /dev/sdc vgextend my_vg /dev/sdc lvextend -L +10G /dev/my_vg/home_lv resize2fs /dev/my_vg/home_lv
LVM Q6: Reduce LVM size?
1. Unmount filesystem umount /dev/vg_data/lv_data 2. Check filesystem e2fsck -f /dev/vg_data/lv_data 3. Shrink filesystem resize2fs /dev/vg_data/lv_data 5G 4. Reduce logical volume lvreduce -L 5G /dev/vg_data/lv_data 5. Mount back mount /dev/vg_data/lv_data /mnt 6. Verify df -h lsblk
LVM Q7: Snapshots?
lvcreate -s -L 1G -n lv_snap /dev/vg/lv
Used for backup/testing.
LVM Q8: Linear vs Striped?
Linear = sequential, Striped = parallel (faster)
LVM Q9: Thin provisioning?
Allocate storage on demand, not upfront.
LVM Q10: Backup & restore config?
Configs stored in /etc/lvm/archive and /etc/lvm/backup
