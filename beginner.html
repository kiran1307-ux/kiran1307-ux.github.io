Beginner Tutorials

Start your Linux & DevOps journey with structured tutorials covering commands, file systems, shell scripting, and foundational DevOps concepts.

1. Introduction to Linux

Linux is an open-source, Unix-like operating system used in servers, desktops, and embedded systems. Known for its stability, security, and flexibility, Linux is widely used in IT and DevOps environments.

2. Linux Distributions

Different distributions (distros) serve different purposes:

  • Ubuntu вЂ“ Beginner-friendly desktop and server.
  • CentOS / Rocky Linux вЂ“ Server-focused, stable environment.
  • Fedora вЂ“ Cutting-edge features and development environment.
  • Debian вЂ“ Extremely stable, foundation for other distros.

3. Terminal Basics

The terminal is used to interact with Linux. Commands follow the format:

command [options] [arguments]

  • pwd вЂ“ Print current directory
  • ls вЂ“ List files in a directory
  • cd <directory> вЂ“ Change directory

4. Linux File System Structure

Linux uses a single-rooted hierarchical file system starting from /. Understanding this structure is crucial for navigating, managing files, and system administration.

  • / вЂ“ Root directory, top of the hierarchy, contains all other directories.
  • /bin вЂ“ Essential user binaries like ls, cp, mv.
  • /sbin вЂ“ System binaries for administration, e.g., shutdown, fsck.
  • /etc вЂ“ System-wide configuration files, e.g., passwd, hosts, fstab.
  • /home вЂ“ User directories for personal files. Example: /home/kiran.
  • /root вЂ“ Home directory for the root user (superuser).
  • /var вЂ“ Variable data like logs, caches, mail. Subdirectories include /var/log, /var/tmp.
  • /usr вЂ“ User-installed programs and libraries, divided into /usr/bin, /usr/lib, /usr/share.
  • /lib вЂ“ Shared libraries for binaries in /bin and /sbin.
  • /tmp вЂ“ Temporary files used by users and applications.
  • /dev вЂ“ Device files representing hardware, e.g., /dev/sda, /dev/tty.
  • /mnt & /media вЂ“ Mount points for disks, USB drives, and removable media.
  • /opt вЂ“ Optional third-party software installations, e.g., /opt/google.
  • /proc вЂ“ Virtual filesystem for kernel and process information, e.g., /proc/cpuinfo.
  • /boot вЂ“ Bootloader and kernel files, e.g., vmlinuz, initrd.img.
  • /srv вЂ“ Data for services provided by the system, e.g., /srv/www for web files.
  • /run вЂ“ Runtime data for system processes, PID files, and sockets.

Linux File Types

Linux treats almost everything as a file. Key file types:

  • Regular File (-) вЂ“ Text, data, or programs (e.g., document.txt).
  • Directory (d) вЂ“ Contains files and subdirectories (e.g., /home/user).
  • Symbolic Link (l) вЂ“ Shortcut to another file/directory (ln -s /path/to/file linkname).
  • Character Device File (c) вЂ“ Devices transferring data character by character (e.g., /dev/tty).
  • Block Device File (b) вЂ“ Devices transferring data in blocks (e.g., /dev/sda1).
  • FIFO / Named Pipe (p) вЂ“ For process communication (mkfifo mypipe).
  • Socket (s) вЂ“ Inter-process network communication (e.g., /var/run/docker.sock).

5. File & Directory Operations

  • ls вЂ“ List files (ls -l for details, ls -a for hidden files)
  • cd <directory> вЂ“ Change directory
  • mkdir <directory> вЂ“ Create directory
  • touch <file> вЂ“ Create empty file
  • cp <source> <destination> вЂ“ Copy files/directories
  • mv <source> <destination> вЂ“ Move/rename files
  • rm <file> вЂ“ Remove file
  • rm -r <directory> вЂ“ Remove directory recursively

6. File Permissions

Linux permissions control who can read, write, or execute files.

  • chmod 755 <file> вЂ“ Change permissions
  • chown user:group <file> вЂ“ Change owner/group
  • ls -l вЂ“ View permissions

7. Viewing & Editing Files

  • cat <file> вЂ“ Display file contents
  • less <file> вЂ“ Scroll through file
  • head <file> вЂ“ Show first 10 lines
  • tail <file> вЂ“ Show last 10 lines (tail -f for real-time)
  • nano <file> вЂ“ Terminal editor
  • vim <file> вЂ“ Advanced editor

8. Package Management

  • Debian/Ubuntu: sudo apt update, sudo apt install <package>
  • RedHat/CentOS: sudo yum install <package>

9. System Info & Networking

  • uname -a вЂ“ System info
  • df -h вЂ“ Disk usage
  • free -h вЂ“ Memory usage
  • top вЂ“ Running processes
  • ping <hostname> вЂ“ Test network
  • ifconfig / ip addr вЂ“ Network interfaces

10. Useful Shortcuts

  • Ctrl + C вЂ“ Stop command
  • Ctrl + Z вЂ“ Pause command
  • Tab вЂ“ Autocomplete
  • Arrow Keys вЂ“ Scroll through history
