Start your Linux & DevOps journey with structured tutorials covering commands, file systems, shell scripting, and foundational DevOps concepts.

1. Introduction to Linux

Linux is an open-source, Unix-like operating system used in servers, desktops, and embedded systems. Known for its stability, security, and flexibility, Linux is widely used in IT and DevOps environments.

2. Linux Distributions

Different distributions (distros) serve different purposes:

  • Ubuntu – Beginner-friendly desktop and server.
  • CentOS / Rocky Linux – Server-focused, stable environment.
  • Fedora – Cutting-edge features and development environment.
  • Debian – Extremely stable, foundation for other distros.

3. Terminal Basics

The terminal is the primary way to interact with Linux. Commands generally follow the format:

command [options] [arguments]

Here are essential commands every beginner should know:

  • pwd – Print current working directory.
  • ls – List files in a directory (ls -l for detailed view, ls -a for hidden files).
  • cd <directory> – Change directory (cd .. to go up one level, cd ~ to go home).
  • mkdir <directory> – Create a new directory.
  • rmdir <directory> – Remove an empty directory.
  • tree – Display directory structure in tree format (install with sudo apt install tree).
  • touch <file> – Create an empty file.
  • cp <source> <destination> – Copy files or directories (cp -r for directories).
  • mv <source> <destination> – Move or rename files.
  • rm <file> – Delete a file.
  • rm -r <directory> – Delete a directory recursively.
  • cat <file> – Display the content of a file.
  • less <file> – View file content page by page.
  • head <file> – View the first 10 lines of a file (head -n 20 for first 20 lines).
  • tail <file> – View the last 10 lines of a file (tail -f to follow updates).
  • grep <pattern> <file> – Search for text in files.
  • find <directory> -name <filename> – Search for files in a directory.
  • uname -a – Show system information.
  • df -h – Display disk usage in human-readable format.
  • free -h – Show memory usage.
  • top – Display running processes and system resource usage.
  • chmod <permissions> <file> – Change file permissions.
  • chown <user:group> <file> – Change file owner and group.
  • Ctrl + C – Stop a running command.
  • Ctrl + Z – Pause a command and send it to background.
  • Ctrl + R – Search command history.
  • Tab – Auto-complete file and folder names.
  • Arrow Keys – Navigate through command history.

💡 Tip: Use man <command> to see the manual page of any command (e.g., man ls).

4. Linux File System Structure

Linux uses a single-rooted hierarchical file system starting from /. Understanding this structure is crucial for navigating, managing files, and system administration.

  • / – Root directory, top of the hierarchy, contains all other directories.
  • /bin – Essential user binaries like ls, cp, mv.
  • /sbin – System binaries for administration, e.g., shutdown, fsck.
  • /etc – System-wide configuration files, e.g., passwd, hosts, fstab.
  • /home – User directories for personal files. Example: /home/kiran.
  • /root – Home directory for the root user (superuser).
  • /var – Variable data like logs, caches, mail. Subdirectories include /var/log, /var/tmp.
  • /usr – User-installed programs and libraries, divided into /usr/bin, /usr/lib, /usr/share.
  • /lib – Shared libraries for binaries in /bin and /sbin.
  • /tmp – Temporary files used by users and applications.
  • /dev – Device files representing hardware, e.g., /dev/sda, /dev/tty.
  • /mnt & /media – Mount points for disks, USB drives, and removable media.
  • /opt – Optional third-party software installations, e.g., /opt/google.
  • /proc – Virtual filesystem for kernel and process information, e.g., /proc/cpuinfo.
  • /boot – Bootloader and kernel files, e.g., vmlinuz, initrd.img.
  • /srv – Data for services provided by the system, e.g., /srv/www for web files.
  • /run – Runtime data for system processes, PID files, and sockets.

Linux File Types

Linux treats almost everything as a file. Key file types:

  • Regular File (-) – Text, data, or programs (e.g., document.txt).
  • Directory (d) – Contains files and subdirectories (e.g., /home/user).
  • Symbolic Link (l) – Shortcut to another file/directory (ln -s /path/to/file linkname).
  • Character Device File (c) – Devices transferring data character by character (e.g., /dev/tty).
  • Block Device File (b) – Devices transferring data in blocks (e.g., /dev/sda1).
  • FIFO / Named Pipe (p) – For process communication (mkfifo mypipe).
  • Socket (s) – Inter-process network communication (e.g., /var/run/docker.sock).

6. File Permissions

Linux permissions control who can read, write, or execute files and directories. Understanding permissions is crucial for system security and proper file management.

Permission Structure

Every file or directory has permissions for three types of users:

  • Owner – The user who owns the file.
  • Group – Users in the same group as the file.
  • Others – All other users.

Permissions are represented as r, w, x:

  • r (read) – Permission to read the file.
  • w (write) – Permission to modify the file.
  • x (execute) – Permission to run the file as a program or script.

Viewing Permissions

Use ls -l to see file permissions:

-rwxr-xr-- 1 kiran devops 1024 Apr 5 2026 myscript.sh

Explanation of -rwxr-xr--:

  • - – Type of file (- for regular file, d for directory, l for link).
  • rwx – Permissions for owner (read, write, execute).
  • r-x – Permissions for group (read and execute).
  • r-- – Permissions for others (read only).

Changing Permissions with chmod

There are two ways to set permissions: symbolic and numeric.

Symbolic Method: 
chmod u+x myscript.sh   # Add execute permission to owner
chmod g-w file.txt      # Remove write permission from group
chmod o=r file.txt      # Set read-only for others
Numeric Method:

Permissions can also be set using numbers:

  • r = 4, w = 2, x = 1
  • Sum the numbers for each user type (owner, group, others)
chmod 755 myscript.sh
# 7 = 4+2+1 -> owner: rwx
# 5 = 4+0+1 -> group: r-x
# 5 = 4+0+1 -> others: r-x

Changing Ownership with chown

You can change the file owner or group using chown:

chown kiran:devops myscript.sh
# Owner = kiran, Group = devops

Directory Permissions

Directories use permissions similarly, but x (execute) allows entering the directory:

  • r – List files in the directory
  • w – Create or delete files in the directory
  • x – Enter the directory

Quick Tips for Beginners

  • Use ls -l frequently to check permissions.
  • Be careful with chmod 777, it gives everyone full access.
  • Use chown to correct file ownership when needed.
  • Combine symbolic and numeric methods to simplify permission management.

11. Shell Scripting

Shell scripting allows you to automate tasks in Linux using the Bash shell. Scripts are a sequence of commands written in a file that can be executed like a program.

Shebang

The first line of a shell script should start with #!/bin/bash to indicate that the script should be run with Bash:

#!/bin/bash
echo "Hello, Linux World!"

Variables

Variables store data that you can use in scripts:

name="Kiran"
echo "Hello $name"

User Input: You can ask the user for input:

echo "Enter your name:"
read name
echo "Hello $name"

Conditionals

Use if, elif, and else to make decisions in scripts:

echo "Enter a number:"
read num

if [ $num -gt 10 ]; then
    echo "Number is greater than 10"
elif [ $num -eq 10 ]; then
    echo "Number is 10"
else
    echo "Number is less than 10"
fi

Loops

Loops help perform repetitive tasks:

For Loop: 
for i in 1 2 3 4 5
do
    echo "Number $i"
done
While Loop: 
count=1
while [ $count -le 5 ]
do
    echo "Count: $count"
    ((count++))
done
Until Loop: 
count=1
until [ $count -gt 5 ]
do
    echo "Count: $count"
    ((count++))
done

Functions

Functions group commands to reuse them in scripts:

greet() {
    echo "Hello $1"
}

greet "Kiran"

Script Execution

Make your script executable and run it:

chmod +x myscript.sh
./myscript.sh

Useful Tips for Beginners

  • Use comments # to explain your code.
  • Test scripts with bash -x script.sh to debug step by step.
  • Use $0, $1, $2 for script and argument variables.
  • Use loops and conditionals to automate repetitive tasks.
  • Combine multiple commands using ;, &&, or ||.

Example Script

#!/bin/bash
# Script to greet a user and check age

echo "Enter your name:"
read name
echo "Hello $name!"

echo "Enter your age:"
read age

if [ $age -ge 18 ]; then
    echo "You are an adult."
else
    echo "You are a minor."
fi

13. Version Control with Git

Git is a distributed version control system that helps track changes to code, collaborate with others, and manage projects efficiently. Learning Git is essential for DevOps and software development.

Installing Git

Install Git on Linux:

# For Debian/Ubuntu
sudo apt update
sudo apt install git

# For CentOS/RHEL
sudo yum install git

Configure your Git username and email:

git config --global user.name "Your Name"
git config --global user.email "you@example.com"
git config --global color.ui auto   # Enable colored output

Initializing a Git Repository

Create a new repository in your project folder:

mkdir myproject
cd myproject
git init
# Creates a .git folder to track changes

Cloning a Repository

Clone an existing repository from GitHub or any remote server:

git clone https://github.com/username/repo.git

Adding and Committing Changes

Track and save changes in Git:

git status               # Check current changes
git add file.txt          # Stage a file
git add .                 # Stage all changes
git commit -m "Initial commit"   # Save changes with a message

Working with Branches

Branches let you work on features or experiments without affecting the main code.

git branch                # List branches
git branch feature1       # Create a new branch
git checkout feature1     # Switch to the branch
git merge feature1        # Merge feature1 into current branch

Working with Remote Repositories

Connect your local repository to a remote repository:

git remote add origin https://github.com/username/repo.git
git push -u origin main     # Push local commits to remote
git pull origin main        # Fetch and merge changes from remote

Viewing Commit History

git log                 # Show commit history
git log --oneline       # Show compact history
git diff                # Show changes not staged
git diff --staged       # Show staged changes

Tips for Beginners

  • Always commit frequently with meaningful messages.
  • Use branches to work on features separately.
  • Pull changes from remote before starting work.
  • Use git status often to avoid mistakes.
  • GitHub, GitLab, and Bitbucket are popular platforms for hosting repositories.

Example Workflow

# Create project and initialize Git
mkdir myproject
cd myproject
git init

# Add files and commit
echo "Hello Linux" > hello.txt
git add hello.txt
git commit -m "Add hello file"

# Push to remote repository
git remote add origin https://github.com/username/myproject.git
git push -u origin main

14. Introduction to DevOps

DevOps is a set of practices that combines software development (Dev) and IT operations (Ops) to shorten the software development lifecycle, improve deployment frequency, and deliver high-quality software reliably.

What is DevOps and Why It Matters

DevOps focuses on collaboration between developers and operations teams. Key benefits include:

  • Faster delivery of software updates and features.
  • Continuous integration and continuous deployment (CI/CD).
  • Improved collaboration and communication between teams.
  • Automated testing, deployment, and infrastructure management.
  • Higher reliability and scalability of applications.

CI/CD Pipeline Basics

A CI/CD pipeline automates building, testing, and deploying code. The main stages include:

  • Continuous Integration (CI): Developers push code frequently, automated tests run to verify changes.
  • Continuous Delivery (CD): Validated code is automatically prepared for deployment to production.
  • Continuous Deployment: Fully automated deployment to production without manual intervention.

Popular tools for CI/CD include Jenkins, GitLab CI/CD, GitHub Actions, and CircleCI.

Introduction to Docker and Containers

Containers package applications with their dependencies so they run consistently across environments.

  • Docker is a platform for creating, managing, and running containers.
  • Containers are lightweight and isolated, unlike virtual machines.
  • Docker images are templates for containers.
  • Key commands:
docker build -t myapp:1.0 .    # Build Docker image
docker run -d -p 8080:80 myapp:1.0  # Run container
docker ps                       # List running containers
docker stop <container_id>      # Stop a container

Infrastructure as Code (IaC)

IaC allows you to manage and provision infrastructure using code, instead of manual setup. Tools include:

  • Terraform – Define infrastructure in declarative configuration files.
  • Ansible – Automate software configuration and deployment.
  • Puppet / Chef – Manage server configurations consistently.

Monitoring and Logging Basics

Monitoring and logging ensure system reliability and help troubleshoot issues:

  • Monitoring tools: Prometheus, Grafana, Nagios – track system health, CPU, memory, and application performance.
  • Logging tools: ELK Stack (Elasticsearch, Logstash, Kibana), Fluentd – collect, analyze, and visualize logs.
  • Alerts notify teams when issues occur, enabling faster response.

Collaboration and Version Control

DevOps heavily relies on collaboration tools and version control:

  • Git – Track code changes and manage branches.
  • Slack / Teams / Jira – Communication and task management.

DevOps Summary

In short, DevOps combines:

  • People: Collaboration across Dev and Ops teams.
  • Processes: Automation of testing, deployment, and infrastructure.
  • Tools: CI/CD, containers, monitoring, IaC, and version control.

💡 Tip: Start small by automating simple tasks with scripts and gradually move to CI/CD pipelines and containerization.
