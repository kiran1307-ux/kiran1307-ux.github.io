Beginner Tutorials

Start your Linux & DevOps journey with structured tutorials covering commands, file systems, shell scripting, and foundational DevOps concepts.

1. Introduction to Linux

Linux is an open-source, Unix-like operating system used in servers, desktops, and embedded systems. Known for its stability, security, and flexibility, Linux is widely used in IT and DevOps environments.

2. Linux Distributions

Different distributions (distros) serve different purposes:

  • Ubuntu – Beginner-friendly desktop and server.
  • CentOS / Rocky Linux – Server-focused, stable environment.
  • Fedora – Cutting-edge features and development environment.
  • Debian – Extremely stable, foundation for other distros.

3. Terminal Basics

The terminal is used to interact with Linux. Commands follow the format:

command [options] [arguments]

  • pwd – Print current directory
  • ls – List files in a directory
  • cd <directory> – Change directory

4. Linux File System Structure

Linux uses a hierarchical file system:

  • / – Root directory
  • /home – User directories
  • /etc – Configuration files
  • /var – Variable data (logs, cache)
  • /usr – Installed programs & libraries
  • /bin – Essential binaries
  • /tmp – Temporary files

5. File & Directory Operations

  • ls – List files (ls -l for details, ls -a for hidden files)
  • cd <directory> – Change directory
  • mkdir <directory> – Create directory
  • touch <file> – Create empty file
  • cp <source> <destination> – Copy files/directories
  • mv <source> <destination> – Move/rename files
  • rm <file> – Remove file
  • rm -r <directory> – Remove directory recursively

6. File Permissions

Linux permissions control who can read, write, or execute files.

  • chmod 755 <file> – Change permissions
  • chown user:group <file> – Change owner/group
  • ls -l – View permissions

7. Viewing & Editing Files

  • cat <file> – Display file contents
  • less <file> – Scroll through file
  • head <file> – Show first 10 lines
  • tail <file> – Show last 10 lines (tail -f for real-time)
  • nano <file> – Terminal editor
  • vim <file> – Advanced editor

8. Package Management

  • Debian/Ubuntu: sudo apt update, sudo apt install <package>
  • RedHat/CentOS: sudo yum install <package>

9. System Info & Networking

  • uname -a – System info
  • df -h – Disk usage
  • free -h – Memory usage
  • top – Running processes
  • ping <hostname> – Test network
  • ifconfig / ip addr – Network interfaces

10. Useful Shortcuts

  • Ctrl + C – Stop command
  • Ctrl + Z – Pause command
  • Tab – Autocomplete
  • Arrow Keys – Scroll through history

4. Version Control with Git

Track code changes and collaborate using Git.

  • Installing Git and configuring user
  • Initializing repositories: git init
  • Cloning, adding, committing: git clone, git add, git commit
  • Branches: git branch, git checkout, git merge
