19. System Monitoring 🔧

Monitoring Linux systems is essential to maintain performance and uptime.

Key Commands:

  • top – Show running processes and CPU/memory usage.
  • htop – Interactive process viewer (needs installation).
  • free -h – Display memory usage.
  • df -h – Check disk space.
  • du -sh – See size of directories.

Tips:

  • Use cron jobs to automate monitoring scripts.
  • Combine with logging for long-term analysis.
20. System Services & Daemons 🖥️

Services run in the background to handle tasks automatically.

Managing Services:

  • systemctl start/stop/restart service_name – Start or stop services.
  • systemctl enable/disable service_name – Auto-start at boot.
  • systemctl status service_name – Check service status.

Common services include nginx, apache2, ssh, and docker.

21. Logs & System Analysis 📊

Logs are key to troubleshooting and understanding system behavior.

Main Log Files:

  • /var/log/syslog – General system messages.
  • /var/log/auth.log – Authentication and login attempts.
  • /var/log/dmesg – Kernel boot messages.
  • /var/log/apache2/error.log – Web server errors.

Useful Commands:

  • tail -f /var/log/syslog – View logs live.
  • grep 'keyword' /var/log/syslog – Filter logs for specific info.
  • journalctl -xe – Systemd journal logs.
22. Environment Variables 🧩

Environment variables store system-wide or user-specific configuration data.

Key Commands:

  • echo $HOME – Show home directory.
  • printenv – List all environment variables.
  • export VAR_NAME=value – Set a new variable temporarily.

Permanent variables can be added in ~/.bashrc or /etc/environment.

23. Networking Tools 🔗

Networking commands help check connectivity and troubleshoot issues.

  • ping google.com – Check network connectivity.
  • ifconfig / ip addr – View network interfaces.
  • netstat -tulnp – See listening ports and services.
  • traceroute google.com – Track packet path.
  • nslookup domain.com – DNS resolution check.
24. Backup & Restore 💾

Regular backups prevent data loss and help recovery from mistakes or failures.

Common Commands:

  • tar -czvf backup.tar.gz /path/to/folder – Compress folder into a backup archive.
  • rsync -av /source /destination – Synchronize files/directories efficiently.
  • scp backup.tar.gz user@remote:/path – Copy backups to remote server.

Tip: Automate backups using cron for daily or weekly tasks.
