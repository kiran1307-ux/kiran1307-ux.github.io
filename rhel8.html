RHEL 8 Hardening Controls

SSH Hardening

File: /etc/ssh/sshd_config

Settings: Disable root login, Protocol 2, Disable empty passwords

Why: Secures SSH access, prevents unauthorized root login, and protects against brute-force attacks.

Journal Logs

Command: journalctl --verify

Why: Ensures logging integrity and allows auditing of system events, helping detect suspicious activity.

Modprobe Hardening

File: /etc/modprobe.d/disable-modules.conf

Settings: Disable unused kernel modules

Why: Reduces attack surface by preventing unnecessary kernel modules from loading.

Password Quality

File: /etc/security/pwquality.conf

Settings: Minimum length, complexity, history enforcement

Why: Enforces strong passwords, reducing risk of brute-force attacks and compromised accounts.

Cron Security

Files: /etc/cron.allow & /etc/cron.deny

Why: Restricts cron job scheduling to authorized users only, preventing unauthorized task execution.

Storage Dump Security

File: /etc/audit/audit.rules

Settings: Monitor critical files and directories

Why: Detects unauthorized access or modification of important files.

Authentication Hardening

File: /etc/pam.d/system-auth

Settings: Enforce password policies and account lockouts

Why: Strengthens user authentication and prevents brute-force attacks.

File Permissions

Command: chmod 600 /etc/shadow

Why: Restricts sensitive files so only authorized users can read/write them.

File Ownership

Command: chown root:root /etc/shadow

Why: Ensures critical files are owned by root to prevent tampering.

SELinux Enforcement

Command: setenforce 1

Why: Enables mandatory access controls to protect the system from unauthorized access.
