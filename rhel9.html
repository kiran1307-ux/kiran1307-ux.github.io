RHEL 9 Hardening Controls

SSH Hardening

File: /etc/ssh/sshd_config

PermitRootLogin no Protocol 2 PermitEmptyPasswords no AllowUsers admin

Why: Secures SSH, disables root login, enforces protocol 2, prevents empty passwords, and allows only authorized users.

Journal Logs

journalctl --verify journalctl --disk-usage

Why: Checks log integrity and disk usage to ensure auditing reliability.

File Ownership

chown root:root /etc/shadow chown root:root /etc/passwd

Why: Ensures sensitive files are owned by root to prevent unauthorized changes.

File Permissions

chmod 600 /etc/shadow chmod 644 /etc/passwd

Why: Restricts access to critical files while keeping necessary files readable.

Authentication Hardening

authselect select sssd --force authselect enable-feature with-faillock

Why: Uses SSSD for centralized authentication and enables account lockout on failed attempts.

Storage Dump Prevention

echo 2 > /proc/sys/kernel/kptr_restrict sysctl -w kernel.dmesg_restrict=1

Why: Hides kernel pointers and restricts dmesg access to prevent information leaks.

Password Quality

/etc/security/pwquality.conf (minlen=14, dcredit=-1, ucredit=-1, ocredit=-1)

Why: Enforces strong password complexity to reduce brute-force attack risk.

Kernel Hardening

sysctl -w net.ipv4.ip_forward=0 sysctl -w net.ipv4.conf.all.accept_redirects=0 sysctl -w net.ipv4.conf.all.send_redirects=0

Why: Disables IP forwarding and ICMP redirects for improved network security.

Modprobe Hardening

echo "install usb-storage /bin/true" > /etc/modprobe.d/disable-usb.conf echo "install firewire-core /bin/true" > /etc/modprobe.d/disable-firewire.conf

Why: Disables unused kernel modules to reduce the system attack surface.
