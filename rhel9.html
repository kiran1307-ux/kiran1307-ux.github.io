SSH Hardening

File: /etc/ssh/sshd_config

Settings: Disable root login, PermitRootLogin no, Use Protocol 2

Why: Protects against remote attacks and ensures encrypted SSH connections.

journald Hardening

Command: journalctl --verify

Why: Maintains secure and reliable logging, essential for auditing and forensic analysis.

File Permissions & Ownership

Command: chmod, chown for critical files

Why: Ensures only authorized users can read/write system files, preventing privilege escalation.

SELinux

Command: setenforce 1 / getenforce

Why: Enforces mandatory access control, isolating processes and preventing unauthorized access.
