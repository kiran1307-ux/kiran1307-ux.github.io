RHEL 9 Hardening Controls

SSH Hardening

File: /etc/ssh/sshd_config

Settings: Disable root login, Use Protocol 2, Disable empty passwords, Use strong ciphers

Why: Protects remote access with modern encryption, prevents root login attacks, and secures SSH communications.

FirewallD Configuration

Command: firewall-cmd --permanent --zone=public --add-service=ssh

Settings: Enable default zone, allow necessary services, block unused ports

Why: Reduces attack surface by allowing only necessary network traffic and securing open ports.

SELinux Enforcement

Command: setenforce 1

Why: Enables mandatory access controls to prevent unauthorized access and privilege escalation.

Password Quality

File: /etc/security/pwquality.conf

Settings: Minimum length, complexity, enforce history, account lockout

Why: Ensures strong passwords and prevents brute-force attacks.

AuditD Configuration

File: /etc/audit/audit.rules

Settings: Monitor sensitive directories and commands

Why: Tracks changes to critical files and detects suspicious activity.

Cron Security

Files: /etc/cron.allow & /etc/cron.deny

Why: Ensures only authorized users can schedule tasks, preventing misuse.

File Permissions

Command: chmod 600 /etc/shadow

Why: Protects sensitive files from unauthorized access.

File Ownership

Command: chown root:root /etc/shadow

Why: Ensures critical files are owned by root, preventing tampering.

Journal Logs

Command: journalctl --verify

Why: Verifies logs for tampering and ensures accurate auditing.
