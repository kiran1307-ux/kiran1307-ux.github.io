Intermediate Linux & DevOps

1. System Administration Deep Dive

At intermediate level, you must understand how services run internally, dependencies, logs, and troubleshooting approach.

systemctl restart nginx
journalctl -xe
ps -ef | grep nginx

Scenario: Service not starting → Check logs, dependency failure, port conflict.

2. Process & Resource Management

Monitor CPU, memory, and processes to identify bottlenecks.

top
htop
kill -9 PID
nice -n 10 process

Scenario: High CPU → Identify process → Kill or renice.

3. Advanced Networking

Understand ports, routing, DNS, and connectivity debugging.

ip a
ss -tulnp
netstat -tulnp
traceroute google.com

Scenario: App not reachable → Check port, firewall, DNS.

4. Disk & LVM Management

Expand storage dynamically using LVM.

pvcreate /dev/sdb
vgextend vg /dev/sdb
lvextend -L +5G /dev/vg/lv
resize2fs /dev/vg/lv

Scenario: Disk full → Extend volume without downtime.

5. Security & Hardening

Secure servers using SSH, firewall, and SELinux.

firewall-cmd --add-service=http
setenforce 1
vi /etc/ssh/sshd_config

Scenario: SSH attack → Disable root login, use key auth.

18. Docker Deep Dive

Docker allows you to package applications with all dependencies into containers, ensuring consistency across environments.

docker build -t myapp .
docker run -d -p 8080:80 myapp
docker exec -it container_id bash
docker logs container_id

Best Practices:

  • Use lightweight base images (alpine)
  • Write small, optimized Dockerfiles
  • Use .dockerignore to reduce image size
  • Avoid running containers as root

Scenario: Application works locally but fails in server → Use Docker image to ensure consistency.

19. Docker Compose (Multi-Container Apps)

Docker Compose helps manage multi-container applications like web + database.

version: '3'
services:
  web:
    image: nginx
    ports:
      - "80:80"
  db:
    image: mysql
    environment:
      MYSQL_ROOT_PASSWORD: root

docker-compose up -d
docker-compose down

Scenario: App requires DB + backend + frontend → Use docker-compose to manage all services.

20. Kubernetes Basics

Kubernetes (K8s) is used to orchestrate containers at scale.

kubectl get pods
kubectl get nodes
kubectl describe pod pod-name
kubectl logs pod-name

Core Concepts:

  • Pod → Smallest deployable unit
  • Deployment → Manages replicas
  • Service → Exposes application

Scenario: Application needs auto-scaling → Deploy using Kubernetes.

21. Kubernetes Deployment Example

apiVersion: apps/v1
kind: Deployment
metadata:
  name: nginx-deployment
spec:
  replicas: 3
  selector:
    matchLabels:
      app: nginx
  template:
    metadata:
      labels:
        app: nginx
    spec:
      containers:
      - name: nginx
        image: nginx
        ports:
        - containerPort: 80

kubectl apply -f deployment.yaml
kubectl scale deployment nginx-deployment --replicas=5

Scenario: High traffic → Scale replicas to handle load.

22. Kubernetes Troubleshooting

kubectl describe pod pod-name
kubectl logs pod-name
kubectl get events

Common Issues:

  • CrashLoopBackOff → App crash
  • ImagePullBackOff → Wrong image
  • Pending → Resource issue

Scenario: Pod not starting → Check logs → Check events → Fix image/config.

23. CI/CD Pipeline Basics

Automate build, test, and deployment using CI/CD pipelines.

Git → Build → Test → Deploy

# Example Jenkins pipeline
pipeline {
  stages {
    stage('Build') {
      steps {
        sh 'docker build -t myapp .'
      }
    }
  }
}

Scenario: Manual deployment errors → Automate using CI/CD.

24. DevOps Best Practices

26. Advanced DevOps Troubleshooting Scenarios

Scenario 1: Website is Down (Production Issue)

Step 1: Check server
uptime

Step 2: Check service
systemctl status nginx

Step 3: Check port
ss -tulnp | grep 80

Step 4: Check logs
journalctl -u nginx
tail -f /var/log/nginx/error.log

Step 5: Check firewall
firewall-cmd --list-all

Root Cause Examples: Service stopped / Port blocked / Config error

Scenario 2: Kubernetes Pod in CrashLoopBackOff

kubectl get pods
kubectl describe pod pod-name
kubectl logs pod-name

Troubleshooting:

  • Check application error in logs
  • Verify environment variables
  • Check resource limits (CPU/Memory)

Root Cause: App crash / Missing config / Dependency failure

Scenario 3: Docker Container Not Starting

docker ps -a
docker logs container_id
docker inspect container_id

Troubleshooting:

  • Check entrypoint/command
  • Verify port conflicts
  • Check missing dependencies

Root Cause: Wrong CMD / Port already used / App failure

Scenario 4: High CPU Usage in Server

top
ps -eo pid,ppid,cmd,%mem,%cpu --sort=-%cpu | head

Troubleshooting:

  • Identify process
  • Restart service or optimize code
  • Use nice/renice

Root Cause: Infinite loop / Memory leak / Heavy traffic

Scenario 5: Disk Full Issue

df -h
du -sh /var/log/*

Troubleshooting:

  • Clean logs
  • Remove old backups
  • Extend LVM

Root Cause: Logs growing / Backup accumulation

Scenario 6: CI/CD Pipeline Failed

Check Jenkins logs
Check build stage logs
Check Docker build errors

Troubleshooting:

  • Syntax error in pipeline
  • Dependency failure
  • Permission issue

Root Cause: Code issue / Config error / Missing credentials

Scenario 7: Application Not Accessible (Network Issue)

ping server_ip
curl http://server_ip:port
telnet server_ip port

Troubleshooting:

  • Check firewall
  • Check service binding (0.0.0.0)
  • Check DNS resolution

Root Cause: Firewall / Wrong bind IP / DNS issue

Scenario 8: Kubernetes Service Not Working

kubectl get svc
kubectl describe svc service-name
kubectl get endpoints

Troubleshooting:

  • Check selector labels
  • Verify pod is running
  • Check port mapping

Root Cause: Label mismatch / No endpoints

Scenario 9: Memory Leak Issue

free -m
top
ps aux --sort=-%mem

Troubleshooting:

  • Identify process consuming memory
  • Restart service
  • Analyze application logs

Root Cause: Poor code / Unreleased memory

Scenario 10: Deployment Rollback

kubectl rollout history deployment
kubectl rollout undo deployment

Troubleshooting:

  • Identify faulty release
  • Rollback quickly

Root Cause: Bad deployment

Scenario 11: SSL Certificate Expired

openssl x509 -in cert.pem -text -noout

Troubleshooting:

  • Renew certificate
  • Restart service

Root Cause: Certificate expiration

Scenario 12: Database Connection Failed

netstat -tulnp | grep 3306
systemctl status mysql

Troubleshooting:

  • Check DB service
  • Verify credentials
  • Check network access

Root Cause: DB down / Wrong config / Firewall

Golden Troubleshooting Approach:

1. Check Logs
2. Check Service
3. Check Resource (CPU/MEM/Disk)
4. Check Network
5. Check Configuration
6. Apply Fix
7. Validate

Key Tip: Always follow → Logs → Metrics → Configuration → Fix
