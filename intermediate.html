Intermediate Linux & DevOps

1. System Administration Deep Dive

At intermediate level, you must understand how services run internally, dependencies, logs, and troubleshooting approach.

systemctl restart nginx
journalctl -xe
ps -ef | grep nginx

Scenario: Service not starting → Check logs, dependency failure, port conflict.

2. Process & Resource Management

Monitor CPU, memory, and processes to identify bottlenecks.

top
htop
kill -9 PID
nice -n 10 process

Scenario: High CPU → Identify process → Kill or renice.

3. Advanced Networking

Understand ports, routing, DNS, and connectivity debugging.

ip a
ss -tulnp
netstat -tulnp
traceroute google.com

Scenario: App not reachable → Check port, firewall, DNS.

4. Disk & LVM Management

Expand storage dynamically using LVM.

pvcreate /dev/sdb
vgextend vg /dev/sdb
lvextend -L +5G /dev/vg/lv
resize2fs /dev/vg/lv

Scenario: Disk full → Extend volume without downtime.

5. Security & Hardening

Secure servers using SSH, firewall, and SELinux.

firewall-cmd --add-service=http
setenforce 1
vi /etc/ssh/sshd_config

Scenario: SSH attack → Disable root login, use key auth.

18. Docker Deep Dive

Docker allows you to package applications with all dependencies into containers, ensuring consistency across environments.

docker build -t myapp .
docker run -d -p 8080:80 myapp
docker exec -it container_id bash
docker logs container_id

Best Practices:

  • Use lightweight base images (alpine)
  • Write small, optimized Dockerfiles
  • Use .dockerignore to reduce image size
  • Avoid running containers as root

Scenario: Application works locally but fails in server → Use Docker image to ensure consistency.

19. Docker Compose (Multi-Container Apps)

Docker Compose helps manage multi-container applications like web + database.

version: '3'
services:
  web:
    image: nginx
    ports:
      - "80:80"
  db:
    image: mysql
    environment:
      MYSQL_ROOT_PASSWORD: root

docker-compose up -d
docker-compose down

Scenario: App requires DB + backend + frontend → Use docker-compose to manage all services.

20. Kubernetes Basics

Kubernetes (K8s) is used to orchestrate containers at scale.

kubectl get pods
kubectl get nodes
kubectl describe pod pod-name
kubectl logs pod-name

Core Concepts:

  • Pod → Smallest deployable unit
  • Deployment → Manages replicas
  • Service → Exposes application

Scenario: Application needs auto-scaling → Deploy using Kubernetes.

21. Kubernetes Deployment Example

apiVersion: apps/v1
kind: Deployment
metadata:
  name: nginx-deployment
spec:
  replicas: 3
  selector:
    matchLabels:
      app: nginx
  template:
    metadata:
      labels:
        app: nginx
    spec:
      containers:
      - name: nginx
        image: nginx
        ports:
        - containerPort: 80

kubectl apply -f deployment.yaml
kubectl scale deployment nginx-deployment --replicas=5

Scenario: High traffic → Scale replicas to handle load.

22. Kubernetes Troubleshooting

kubectl describe pod pod-name
kubectl logs pod-name
kubectl get events

Common Issues:

  • CrashLoopBackOff → App crash
  • ImagePullBackOff → Wrong image
  • Pending → Resource issue

Scenario: Pod not starting → Check logs → Check events → Fix image/config.

23. CI/CD Pipeline Basics

Automate build, test, and deployment using CI/CD pipelines.

Git → Build → Test → Deploy

# Example Jenkins pipeline
pipeline {
  stages {
    stage('Build') {
      steps {
        sh 'docker build -t myapp .'
      }
    }
  }
}

Scenario: Manual deployment errors → Automate using CI/CD.

24. DevOps Best Practices

  • Use Infrastructure as Code (Terraform, Ansible)
  • Automate everything (CI/CD pipelines)
  • Monitor systems (Prometheus, Grafana)
  • Implement logging (ELK Stack)
  • Follow security best practices

Scenario: Production outage → Monitoring alert → Quick rollback using CI/CD.
