Intermediate Linux & DevOps Tutorials

1. Linux System Administration

System administration involves managing services, monitoring processes, and troubleshooting issues in real time.

systemctl status nginx
journalctl -u nginx
top

Scenario: Website is down.

  • Check service: systemctl status nginx
  • Check logs: journalctl -u nginx
  • Restart service: systemctl restart nginx

2. Security & Hardening

Security ensures system protection using firewalls, SELinux, and SSH configurations.

firewall-cmd --add-port=80/tcp --permanent
firewall-cmd --reload
setenforce 1

Scenario: Application is running but not accessible.

  • Check firewall rules
  • Allow required port (80/443)
  • Verify SELinux status

3. Advanced Shell Scripting

Used to automate tasks with logic, error handling, and logging.

#!/bin/bash
LOGFILE="script.log"

if [ $? -ne 0 ]; then
  echo "Error occurred" >> $LOGFILE
fi

Scenario: Automate log cleanup daily.

  • Create script to delete old logs
  • Schedule using cron

4. Git Branching & Collaboration

Allows teams to work on features independently.

git checkout -b feature-login
git add .
git commit -m "Added login feature"
git push origin feature-login

Scenario: Multiple developers working on same project.

  • Create feature branch
  • Merge into main after testing
  • Resolve conflicts if any

5. Docker Compose & Multi-Container Apps

Run multiple containers (web + DB) together.

version: '3'
services:
  web:
    image: nginx
  db:
    image: mysql

Scenario: Deploy web app with database.

  • Use docker-compose
  • Link containers via network

6. Kubernetes Deployment

Deploy and manage applications at scale.

kubectl apply -f deployment.yaml
kubectl get pods
kubectl scale deployment app --replicas=3

Scenario: Traffic increased suddenly.

  • Scale pods from 2 → 5
  • Monitor using kubectl get pods

7. Infrastructure Automation (Ansible)

Automate configuration across multiple servers.

- hosts: web
  tasks:
    - name: Install nginx
      apt:
        name: nginx
        state: present

Scenario: Install nginx on 50 servers.

  • Write playbook
  • Run once instead of manual install

8. CI/CD Pipelines

Automate build, test, and deployment.

pipeline {
  stages {
    stage('Build') {
      steps { echo 'Building...' }
    }
  }
}

Scenario: Developer pushes code.

  • Pipeline triggers automatically
  • Build → Test → Deploy

9. Disk & LVM Management

Manage and extend storage dynamically.

lvextend -L +5G /dev/vg/lv
resize2fs /dev/vg/lv

Scenario: /var is full.

  • Extend logical volume
  • Resize filesystem

10. Networking Troubleshooting

Diagnose connectivity and port issues.

ping google.com
ss -tulnp
curl http://localhost

Scenario: Application not reachable.

  • Check port listening
  • Test connectivity

11. Log Analysis

Logs help identify root cause of issues.

tail -f /var/log/messages
grep ERROR /var/log/syslog

Scenario: Service failed unexpectedly.

  • Check logs for errors
  • Identify root cause

12. Performance Monitoring

Monitor system resources.

top
htop
iostat
vmstat

Scenario: Server is slow.

  • Check CPU usage
  • Identify high process

13. Backup & Restore

Ensure data safety.

tar -cvf backup.tar /data
rsync -av /source /backup

Scenario: Before patching server.

  • Take backup
  • Restore if failure occurs

14. User & Access Management

Manage users and permissions securely.

useradd devops
passwd devops
usermod -aG sudo devops

Scenario: New team member joins.

  • Create user
  • Assign sudo access
